# Architecture Notes — Sprint Y Blog Infrastructure

## Codebase Structure
- Next.js 15 App Router: all pages in src/app/
- Components: src/components/
- Utilities: src/lib/
- Tests: tests/ (mirrors src/ structure)
- Build tool: next build (Turbopack)
- Test runner: vitest
- Type checking: tsc --noEmit

## Design System Patterns
- Primary color: indigo-600 (hover: indigo-700)
- Background: #F7F7F5 (light beige)
- Cards: bg-white with rounded-2xl shadow-sm
- Responsive grids: grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3
- Spacing: consistent use of px-4 sm:px-6 lg:px-8 for containers
- Max width container: max-w-7xl mx-auto

## Component Patterns
- All interactive components use 'use client' directive
- Navigation: src/components/nav-bar.tsx — navLinks array at line 17
- Footer: src/components/footer.tsx
- Auth provider pattern: useAuth hook from @/components/auth-provider

## Routing
- App Router conventions: page.tsx, layout.tsx
- Dynamic routes: [slug] directories
- API routes: src/app/api/*/route.ts (Next.js 15 App Router)

## Navigation Integration
- NavBar links defined in navLinks array (line 17 of nav-bar.tsx)
- Add blog link: { href: '/blog', label: 'Blog' }
- Mobile menu support built-in (responsive)

## Homepage Structure (src/app/home-client.tsx)
- Client component with useState hooks
- Sections: Hero → How It Works (id="how-it-works") → Features → FAQ
- Insert blog section after hero, before FAQ (around line 120)
- Uses motion.div for animations (framer-motion)
- Stats fetched from /api/stats

## Sitemap Pattern (src/app/sitemap.ts)
- Returns MetadataRoute.Sitemap array
- baseUrl: https://aimeajob.com
- Priority scale: 1.0 (home) → 0.9 (upload) → 0.8 (pricing) → etc.
- Blog should be priority 0.8, articles 0.7

## Test Conventions
- Test files in tests/ mirror src/ structure
- Use vitest (NOT jest)
- Mocking pattern: vi.mock('next/navigation', () => ({ ... }))
- Component tests: render from @testing-library/react
- Screen queries: screen.getByText, screen.getAllByText
- Assertions: expect(value).toBeDefined(), expect(value).toBe(...)
- Snapshot tests for complex layouts

## Error Handling
- No explicit try/catch in most client components
- API routes use try/catch with JSON error responses
- Error boundaries: src/app/error.tsx

## TypeScript Rules
- Strict mode enabled
- No any, no ts-ignore
- All functions and components typed
- Interfaces for complex props
- Import aliases: @/components, @/lib, @/app

## Environment
- .env required for build (DATABASE_URL, LLM config)
- .env.example provides template
- Supabase for auth and DB

## Build Requirements
- Must have DATABASE_URL env var (even if placeholder)
- Build produces static pages where possible
- Dynamic routes marked with ƒ in build output
- Static routes marked with ○

## Critical for Blog Sprint
1. MDX config must go in next.config.js (not next.config.ts)
2. Use @next/mdx with createMDX wrapper
3. pageExtensions: ['js','jsx','ts','tsx','md','mdx']
4. Content directory: content/blog/
5. Gray-matter for frontmatter parsing
6. Blog listing: read MDX files, parse frontmatter, sort by date
7. Build must pass after MDX setup before continuing to other stories

## Naming Conventions
- Files: kebab-case (nav-bar.tsx, home-client.tsx)
- Components: PascalCase (NavBar, HomeClient)
- Functions: camelCase (getStats, fetchJobs)
- Types/Interfaces: PascalCase (Stats, Job)
- CSS classes: Tailwind utility classes

## Import Patterns
- Next.js: import Link from 'next/link'
- Components: import { NavBar } from '@/components/nav-bar'
- Utils: import { db } from '@/lib/db'
- Types: import type { Stats } from '@/lib/types'
