# Architecture Notes for AI Job Matcher

## Route Structure
- All API routes in `src/app/api/*/route.ts` follow Next.js 16 App Router pattern
- Routes use `NextRequest` and `NextResponse` from 'next/server'
- Authentication via X-Restore-Token header, validated with `validateRestoreToken(profileId, token)`
- Routes export `runtime = 'nodejs'` for serverless compatibility

## Error Handling Pattern
Standard try-catch with typed responses:
```typescript
try {
  // logic
  return NextResponse.json({ data: ... });
} catch (error: unknown) {
  if (error instanceof Response) return error;
  if (error instanceof Error && error.message === 'Profile not found') {
    return NextResponse.json({ error: 'Profile not found' }, { status: 404 });
  }
  return NextResponse.json({ error: 'Internal server error' }, { status: 500 });
}
```

## Database Access
- Uses Drizzle ORM with PostgreSQL
- DB client imported from `@/lib/db/index`
- Schema in `@/lib/db/schema` (PROTECTED — DO NOT MODIFY)
- Queries use `eq`, `gt`, `desc`, `sql` from 'drizzle-orm'
- Tables: profiles, jobs, searches, searchResults

## Matching Engine (src/lib/matching/engine.ts)
- `findMatches(profileId, options?)` returns `MatchResult`
- Scoring: 8 criteria (years_experience, education_level, field_of_study, sphere_of_expertise, seniority_level, languages, industry, key_skills)
- Filter threshold: score >= 5
- Default sort: score DESC, then posted_at DESC NULLS LAST
- Returns top 50 results
- Creates searches and searchResults records in DB

## Test Organization
- Tests use Vitest with @testing-library/react for components
- Test files co-located with source: `*.test.ts` or `*.test.tsx`
- API route tests in `tests/api/`
- Lib tests in `tests/lib/` and `src/lib/*/`
- Component tests in `src/app/*/page.test.tsx` or `tests/components/`
- Baseline: 140 tests passing

## Utility Functions
- `formatLabel(key: string)`: converts snake_case to Title Case
- `formatSalary(min, max, currency)`: formats salary ranges (e.g., "USD 50k–80k")
- `getAvatarColor(name)`: deterministic color hash for company avatars
- Date formatting: uses `toLocaleDateString()` for display

## Naming Conventions
- React components: PascalCase (e.g., `JobCard`, `ResultsContent`)
- Files: kebab-case (e.g., `extract-text.ts`, `batch-classify.ts`)
- Functions: camelCase (e.g., `findMatches`, `calculateMatch`)
- Database fields: snake_case (e.g., `salary_min`, `posted_at`)
- Component props: camelCase
- CSS classes: Tailwind utility classes

## Frontend Patterns
- All pages in `src/app/` are 'use client' React Server Components become Client Components
- State management: useState + useEffect for data fetching
- URL params: useSearchParams() from 'next/navigation'
- Loading states: skeleton components from '@/components/ui/skeleton'
- Animations: framer-motion with AnimatePresence
- Styling: Tailwind CSS v4 with utility-first approach

## Environment
- Required: DATABASE_URL, LLM_PROVIDER, LLM_MODEL, OPENROUTER_API_KEY
- .env file created with placeholders for build (actual DB not required for type checking)

## Import Paths
- Use '@/' alias for absolute imports from `src/` (e.g., '@/lib/db/schema')
- External deps: npm packages from package.json
- No relative imports crossing major boundaries (prefer '@/' alias)
