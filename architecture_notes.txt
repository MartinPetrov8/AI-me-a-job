# Codebase Architecture Notes — Sprint G

## Directory Structure
- **src/app/**: Next.js 15 App Router pages (restore/, upload/, results/, etc.)
- **src/components/**: Shared React components (tag-input, ui/*)
- **src/lib/**: Business logic, DB, auth, LLM integration
  - auth/: validateRestoreToken helper (used across API routes)
  - db/: Drizzle ORM schema + connection
  - llm/: Claude/OpenRouter client wrappers
  - matching/: Job matching engine
  - ingestion/: Job scrapers (Adzuna, Jooble, DevBG, JobsBG)
- **src/app/api/**: API routes (upload, restore, search, save, preferences, etc.)

## Error Handling Patterns
- API routes return { error: string, details?: any } on failure
- Client-side components show error state with red text (text-red-600)
- HTTP status codes: 200 OK, 400 validation error, 401 unauthorized, 404 not found, 500 server error
- Errors are logged to console with context (e.g., `[pipeline] Ingestion failed`)

## Authentication Flow
- Users upload CV → receive restore_token (long-lived, never expires)
- restore_token stored in localStorage and sent via X-Restore-Token header
- validateRestoreToken() checks DB for matching users.restoreToken
- No password auth — token-based only

## Testing Patterns
- Vitest + React Testing Library
- API route tests: mock fetch, validate response shapes
- Component tests: render, fireEvent, expect screen.getBy*
- Failing tests: mostly test infrastructure issues (multiple "Edit filters" buttons, missing DB_URL in test env)

## Design System
- Tailwind CSS utility classes
- Primary brand: indigo-600 (buttons, links, active states)
- Background: bg-[#F7F7F5] (off-white)
- Cards: bg-white rounded-2xl shadow-sm border border-gray-100 p-8
- Inputs: border-gray-200 rounded-xl focus:ring-indigo-500
- Typography: text-gray-900 (headings), text-gray-500 (body/subtext)

## Naming Conventions
- React components: PascalCase (RestorePage, TagInput)
- Files: kebab-case (restore/page.tsx, validate-restore-token.ts)
- API routes: route.ts (REST conventions)
- DB tables: snake_case (users, profiles, jobs, matches)

## Utility Functions to Reuse
- `validateRestoreToken(token)` → returns user or throws
- `extractCV(fileBuffer)` → parses PDF/DOCX into structured data
- `matchJobs(profile, jobs)` → scoring algorithm
- `classifyJob(job)` → LLM-based job categorization

## Key Constraints for This Sprint
- NO changes to API routes (client-side UI only)
- NO new npm packages
- NO DB schema changes
- Preserve existing restore flow logic (localStorage + X-Restore-Token)
