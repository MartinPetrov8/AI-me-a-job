# Codebase Architecture Patterns

## API Routes (src/app/api/*/route.ts)
- **Structure**: App Router convention — each route is a `route.ts` file exporting async handler functions (GET, POST, etc.)
- **Runtime**: Export `export const runtime = 'nodejs';` for Node.js runtime (required for DB access, LLM calls)
- **Request/Response**: Use NextRequest and NextResponse from 'next/server'
- **Error handling**: 
  - Validate input with Zod schemas using `.safeParse()` — return 400 with `{ errors: [...] }` structure
  - Catch errors and return 500 with `{ errors: [{ code: 'INTERNAL_ERROR', message: '...' }] }`
  - Use status codes: 200 (success), 400 (validation), 404 (not found), 409 (conflict), 500 (internal error)
- **Response format**: `{ data: {...} }` on success, `{ errors: [{code, message}] }` on error
- **Example**: src/app/api/save/route.ts — POST handler with Zod validation, DB interaction, conflict detection

## LLM Integration (src/lib/llm/)
- **Client**: `callLLM()` in client.ts — generic wrapper supporting multiple providers (OpenRouter, Anthropic, future: Ollama)
- **Provider selection**: Uses LLM_PROVIDER env var ('openrouter' | 'anthropic')
- **Prompts**: System prompts in prompts.ts — CV_EXTRACTION_SYSTEM_PROMPT and JOB_CLASSIFICATION_SYSTEM_PROMPT
- **Retry logic**: MAX_RETRIES = 2, TIMEOUT_MS = 30000 (30s) — retry on AbortError and SyntaxError (bad JSON)
- **JSON parsing**: Strip markdown code fences before parsing: `.replace(/^```(?:json)?\s*/i, '').replace(/```\s*$/i, '')`
- **Validation**: Use `validateValue()` or `fuzzyNormalize()` to enforce enum constraints — set to null if value doesn't match
- **Classification**: 
  - CV extraction (extract-cv.ts) uses callLLM() → OpenRouter/Qwen by default
  - Job classification (classify-job.ts) currently uses callLLM() → to be changed to Anthropic SDK directly
- **Graceful degradation**: If all retries fail, return null fields instead of throwing (jobs stored but unclassified)

## Database (src/lib/db/)
- **ORM**: Drizzle ORM with Postgres (Supabase)
- **Schema**: src/lib/db/schema.ts — tables: jobs, users, profiles
- **Query patterns**: 
  - `db.select().from(table).where(condition).limit(n)`
  - `db.insert(table).values({...}).onConflictDoUpdate({target, set})`
  - `db.update(table).set({...}).where(condition)`
  - Use `eq`, `ne`, `and`, `or`, `isNull`, `lt` from drizzle-orm
- **Jobs table**: Has all classification fields (yearsExperience, educationLevel, etc.) + classifiedAt timestamp

## Ingestion (src/lib/ingestion/)
- **Sources**: Adzuna (7 countries) and Jooble (Eastern Europe) — see adzuna.ts, jooble.ts
- **Orchestration**: `ingestAllSources()` in ingest.ts — fetches from all sources, upserts jobs, runs cleanup
- **Deduplication**: `normalizeUrl()` and `computeContentHash()` in dedup.ts
- **Upsert strategy**: `onConflictDoUpdate` on [source, externalId] — updates existing jobs if external data changed
- **Cleanup**: Runs AFTER all sources ingested (prevents race conditions)
  - Delete jobs older than 30 days
  - Truncate descriptions for jobs older than 7 days
- **Return type**: `IngestionResult[]` — array of {source, fetched, new, errors, deleted}

## Testing (tests/)
- **Runner**: Vitest (v4.0.18) — run with `npm run test` or `npm run test:watch`
- **Mocking**: Use `vi.mock()` at top of test file to mock modules
- **DB mocking**: Mock fluent query builder chains (see tests/api/save.test.ts)
- **LLM mocking**: Mock callLLM() to return JSON strings (see tests/lib/llm/classify-job.test.ts)
- **Structure**: tests/ mirrors src/ structure
- **Coverage**: 16 test files, 102 tests — focus on lib/ and api/ routes

## Naming Conventions
- **Files**: kebab-case (classify-job.ts, batch-classify.ts)
- **Functions**: camelCase (classifyJob, extractCvCriteria, ingestAllSources)
- **Types/Interfaces**: PascalCase (ClassifiedCriteria, ExtractedCriteria, IngestionResult)
- **Database fields**: snake_case in schema (years_experience, education_level) — matches DB column names
- **API fields**: camelCase in request/response (yearsExperience in TypeScript, years_experience in DB)

## Utility Functions to Reuse
- **src/lib/llm/classify-job.ts**: `validateValue()` — validates enum values with exact match
- **src/lib/llm/extract-cv.ts**: `fuzzyNormalize()` — validates enum values with alias map + partial matching
- **src/lib/ingestion/dedup.ts**: `normalizeUrl()`, `computeContentHash()` — deduplication helpers
- **src/lib/validation.ts**: Zod schemas for common types

## Environment Variables
- DATABASE_URL: Postgres connection string (required for build)
- LLM_PROVIDER: 'openrouter' | 'anthropic'
- LLM_MODEL: model identifier (e.g. 'claude-haiku-4-5-20251001')
- OPENROUTER_API_KEY: for OpenRouter
- ANTHROPIC_API_KEY: for Anthropic direct
- CRON_SECRET: for Vercel cron auth (custom env var)
- ADZUNA_APP_ID, ADZUNA_APP_KEY, JOOBLE_API_KEY: ingestion APIs
